# 产业招投平台

产业招投平台是面向产业招商、投资管理与投后监测的一体化业务平台。系统以产业分析、目标企业筛选、投管池管理和投后跟踪为核心流程，帮助招商与投资团队完成从“发现机会”到“持续管理”的闭环。

本文档用于说明平台的产品定位、技术栈、路由结构与页面职责，便于前端开发、原型演示和后续需求沟通。

## 技术栈

项目采用 React + TypeScript 全家桶技术栈进行开发，约定使用以下技术与第三方包：

| 类型 | 技术 |
| --- | --- |
| 前端框架 | React |
| 开发语言 | TypeScript |
| 路由管理 | react-router-dom |
| 状态管理 | Zustand |
| 网络请求 | Axios |
| UI 组件 | Ant Design |
| 图表展示 | ECharts、echarts-for-react |
| 树状图 / 脑图 / 关系图谱 | AntV G6 |
| 流程图 / 可编辑关系图 | AntV X6 |
| 构建工具 | Vite |
| 代码规范 | ESLint、Prettier |

> 具体依赖以项目实际 `package.json` 为准。

## 核心依赖约定

为避免同类能力重复引入多个库，项目统一使用以下第三方包：

| 场景 | 约定包 | 用途 |
| --- | --- | --- |
| 路由 | `react-router-dom` | 登录、主页、产业详情、公司详情、投管池、投后等页面路由 |
| UI 组件 | `antd` | 表单、按钮、菜单、Tabs、Drawer、Modal、Table、Select、Tooltip、Tag、Card 等基础组件 |
| 状态管理 | `zustand` | 登录态、用户信息、全局筛选条件、跨页面共享数据 |
| 请求 | `axios` | 后续接口请求封装、拦截器、错误处理 |
| 普通图表 | `echarts`、`echarts-for-react` | 柱状图、折线图、饼图、环形图、桑基图等数据可视化 |
| 产业图谱 / 树状图 / 脑图 | `@antv/g6` | 产业链树状图谱、脑图、节点 hover、节点展开收起、节点关系渲染 |
| 复杂关系编辑 | `@antv/x6` | 如后续需要可拖拽、可编辑的流程图或关系图编辑器 |

组件使用约定：

- 页面路由统一使用 `react-router-dom` 的 `createBrowserRouter` 或 `Routes` 方案，路由参数包括 `industryChainId`、`companyId`。
- 基础 UI 统一使用 `antd`，公司简介使用 `Drawer`，顶部页签使用 `Tabs`，表单筛选使用 `Form`、`Input`、`Select`，列表和企业信息使用 `Table`、`Descriptions`、`Tag`。
- 产业趋势、企业数量趋势、营业收入趋势、投融资分布等普通统计图统一使用 `echarts`。
- 产业图谱、产业链强弱分析、树状图、脑图优先使用 `@antv/g6`，避免手写复杂 SVG / Canvas 图谱逻辑。
- 只有在需要节点拖拽编辑、流程编排、关系图编辑时才引入 `@antv/x6`。

## 平台模块

平台包含登录、主页、投管池、投后三个一级业务页面，并包含产业详情、公司详情等下钻页面。其中主页、产业详情、公司详情、投管池、投后属于登录后的业务区，共用同一个页面框架与公共头部。

### 登录

登录页用于完成用户身份认证，进入系统后跳转至主页。

主要内容：

- 账号、密码输入
- 登录按钮
- 登录异常提示
- 登录成功后的路由跳转

建议路由：

```text
/login
```

### 主页

主页用于展示产业链检索与产业专题入口，是用户进入系统后的默认页面。页面采用左右两栏布局：左侧为搜索和产业菜单，右侧为当前选中产业或区域规划的正文内容。

主要内容：

- 产业链名称关键词搜索
- 精选产业专题列表
- 区域重点产业列表
- 当前选中项的正文内容
- 产业链卡片、区域产业规划、重点产业条目展示

建议路由：

```text
/home
```

首页左侧内容：

- 搜索框：用于按产业链名称关键词检索，提示文案为“请输入产业链名称关键”。
- 精选产业专题：展示先进制造产业、绿色低碳产业、战略新兴产业、大消费产业、现代服务产业、生命健康产业、信息科技产业、文娱传媒产业、金融地产产业、传统产业等专题。
- 区域重点产业：展示十四五规划产业、十五五规划产业等区域产业规划入口。

首页右侧正文需要根据左侧点击项切换：

- 点击“精选产业专题”下的条目时，右侧展示该专题的简介和产业链卡片列表。例如点击“先进制造产业”，右侧展示先进制造产业说明，并以卡片形式展示新材料产业链、新能源汽车产业链、生物医药产业链、医疗器械产业链、计算机产业链、通信产业链等。
- 点击“区域重点产业”下的条目时，右侧展示区域规划类正文。例如点击“十四五规划产业”，右侧展示“南通市 · 十四五规划产业”，并按产业方向展示战略性新兴产业、节能环保产业、数字创意产业、生产性服务业、生活性服务业等内容。
- 两类入口的右侧内容结构不同，需要分别建模和渲染，不能复用同一套正文模板。
- 左侧当前选中项需要有明显激活态，右侧正文随选中项实时刷新。
- 右侧正文中的产业链卡片或产业链标签可点击，点击后跳转至产业详情页。

### 产业详情

产业详情页是从主页产业链入口下钻后的二级页面，用于展示单个产业链的产业概况、产业图谱和产业链强弱分析。

建议路由：

```text
/industries/:industryChainId
```

主要内容：

- 左上方产业链名称选择器
- 顶部页签：产业概况、产业图谱、产业链强弱分析
- 产业概况：展示产业链简介、产业定义、上下游范围、核心环节和基础信息
- 产业图谱：展示产业链上游、中游、下游关系图
- 产业链强弱分析：展示产业链强补延图谱，标识强节点、中节点、弱节点、空节点及补链、强链建议环节
- 返回产业列表入口

交互要求：

- 从主页点击产业链卡片或产业链标签后，进入对应产业链详情页。
- 详情页左上方展示当前产业链名称，例如“电子信息产业链”。
- 点击左上方产业链名称选择器，可切换到其他产业链。
- 切换产业链后，详情页内容需要同步刷新，并更新当前路由参数或查询参数。
- 点击“返回产业列表”回到进入详情前的产业列表上下文；如果无法识别来源，默认返回 `/home`。
- 顶部页签切换时，仅切换当前产业链下的内容模块，不改变当前产业链。

产业图谱要求：

- 页面标题建议为“产业链图谱”。
- 主体区域使用 `@antv/g6` 以树状图谱展示产业链层级关系，支持从核心产业链向上下游子节点展开。
- 树子节点鼠标移入时，需要展示相关公司浮层。
- 公司浮层至少展示 2 条相关公司名称，并按序号排列。
- 公司浮层中的公司名称可点击，点击后在当前页面打开公司简介抽屉。
- 公司浮层底部提供“查看更多”入口，用于进入更多相关企业列表或后续企业检索结果页。
- 鼠标移出节点或浮层区域后，浮层关闭。
- 公司简介抽屉打开后，产业图谱页面保持在当前状态，关闭抽屉后回到原图谱位置和节点展开状态。
- 公司简介抽屉用于展示公司基础画像，包括公司名称、企业标签、风险提示、法定代表人、成立日期、注册资本、联系方式、联系地址、所属重点产业链、所属国标行业等。
- 公司简介抽屉可展示企业评价信息，包括企业综合评分、企业科创评分、企业商机价值、企业触达价值等模块。
- 公司简介抽屉顶部提供“查看企业洞察详情”入口，点击后跳转至公司详情页。
- 节点可展示展开、收起状态，支持查看全部节点或当前节点路径。
- 右侧可展示产业发展趋势，包括企业数量趋势、营业收入趋势、投融资轮次分布等图表；右侧趋势信息不影响树节点 hover 交互。
- 树节点数据需要绑定相关公司列表，避免只展示静态节点名称。

产业链强弱分析要求：

- 页面标题建议为“产业链强补延图谱”。
- 优先使用 `@antv/g6` 从左到右展示产业链节点和节点之间的上下游关系，可采用桑基图、树状链路图或自定义图谱组件。
- 节点需要按强弱状态区分颜色：强节点、中节点、弱节点、空节点。
- 顶部图例需要展示强节点、中节点、弱节点、空节点的颜色说明。
- 支持展示补链建议环节、强链建议环节，可使用勾选框控制对应建议节点是否高亮。
- 节点可点击或悬浮展示详情，包括节点名称、所属层级、强弱状态、关联企业数量、问题说明和建议动作。
- 图谱中可按基础层、技术层、应用层等层级组织节点，也可按上游、中游、下游组织节点，具体以产业链数据结构为准。
- 弱节点和空节点应作为重点关注对象，支持在视觉上突出，便于识别补链方向。

### 公司详情

公司详情页是从公司简介抽屉进入的完整企业洞察页面，用于展示企业基础信息、经营情况、风险、产业归属、评价和同业分析。

建议路由：

```text
/companies/:companyId
```

主要内容：

- 企业顶部信息区：企业 Logo、企业名称、经营状态、股票信息、企业资质标签、下载报告、关注入口
- 一级页签：基本信息、资本市场、经营信息、经营风险、司法风险、行业信息、知识产权、企业资讯、图谱中心、企业评价、同业分析、电商详情
- 基本信息二级页签：企业基本信息、股东信息、主要人员、工商变更、股权投资、受益所有人、分支机构、工商年报
- 企业基本信息：公司名称、统一社会信用代码、成立日期、实际控制人、企业类型、注册地址、联系方式、企业网址、所属行业、经营范围、主营业务等
- 企业评价：企业综合评分、企业科创评分、企业商机价值、企业触达价值等

交互要求：

- 从公司简介抽屉点击“查看企业洞察详情”后，跳转至 `/companies/:companyId`。
- 公司详情页根据 `companyId` 拉取或匹配对应企业数据。
- 公司详情页属于业务下钻页面，不作为公共头部中的一级导航项。
- 如果从产业图谱进入公司详情，返回时应优先回到原产业详情页和产业图谱上下文；如果无法识别来源，默认返回 `/home`。
- 点击“下载报告”可触发报告下载或展示下载提示。
- 点击“关注”可切换企业关注状态。

### 投管池

投管池用于管理招商和投资过程中的目标企业、储备项目与跟进状态。

主要内容：

- 企业 / 项目列表
- 项目阶段、所属产业、负责人等筛选条件
- 项目详情入口
- 加入、移出或更新投管池状态
- 投前、投中相关材料沉淀

建议路由：

```text
/investment-pool
```

### 投后

投后模块用于跟踪已投项目的经营、财务、舆情、风险和估值变化，辅助投后管理。

主要内容：

- 已投企业列表
- 投后指标看板
- 风险预警
- 经营与财务趋势
- 投后跟进记录

建议路由：

```text
/post-investment
```

## 路由结构

平台可按“登录页 + 业务布局”组织路由：

```text
/
├── /login                  登录页
└── /                       业务布局 MainLayout
    ├── /home               主页
    ├── /industries/:industryChainId  产业详情
    ├── /companies/:companyId         公司详情
    ├── /investment-pool    投管池
    └── /post-investment    投后
```

路由说明：

- `/login` 不展示业务公共头部。
- `/home`、`/industries/:industryChainId`、`/companies/:companyId`、`/investment-pool`、`/post-investment` 展示公共头部。
- 未登录访问业务路由时，应重定向至 `/login`。
- 登录成功后，默认跳转至 `/home`。
- 产业详情页和公司详情页属于业务下钻页面，不作为公共头部中的一级导航项。

## 公共头部

主页、投管池、投后三个业务页面共用公共头部。头部用于承载平台名称、一级导航和用户操作入口。

当前公共头部需要展示的一级导航：

- 主页
- 投管池

建议头部结构：

```text
┌──────────────────────────────────────────────┐
│ 产业招投平台        主页    投管池      用户信息 │
└──────────────────────────────────────────────┘
```

交互要求：

- 点击“主页”跳转至 `/home`。
- 点击“投管池”跳转至 `/investment-pool`。
- 当前所在导航需要有激活态。
- 投后页面仍使用同一公共头部，但当前导航可按业务需要不展示或后续补充“投后”入口。

## 页面布局建议

业务页面建议统一采用以下布局：

```text
MainLayout
├── Header 公共头部
└── Content 页面内容区
    ├── HomePage
    ├── IndustryDetailPage
    ├── CompanyDetailPage
    ├── InvestmentPoolPage
    └── PostInvestmentPage
```

登录页独立于 `MainLayout`：

```text
LoginPage
```

首页内部布局建议：

```text
HomePage
├── Sidebar
│   ├── SearchInput
│   ├── FeaturedIndustryMenu
│   └── RegionalIndustryMenu
└── HomeContent
    ├── FeaturedIndustryContent
    └── RegionalIndustryContent
```

内容渲染说明：

- `FeaturedIndustryContent` 用于渲染精选产业专题正文，重点是专题介绍和产业链卡片网格。
- `RegionalIndustryContent` 用于渲染区域重点产业正文，重点是地区、规划周期、产业方向分组和产业链标签。
- `HomeContent` 根据左侧菜单选中类型决定渲染哪个正文组件。
- 产业链卡片和产业链标签统一绑定产业链 ID，点击后跳转至 `IndustryDetailPage`。

产业详情页内部布局建议：

```text
IndustryDetailPage
├── IndustryDetailHeader
│   ├── IndustryChainSelector
│   ├── IndustryTabs
│   └── BackToIndustryList
└── IndustryDetailContent
    ├── IndustryOverview
    ├── IndustryMap
    │   └── CompanyProfileDrawer
    └── IndustryStrengthAnalysis
```

内容渲染说明：

- `IndustryChainSelector` 位于页面左上方，用于切换不同产业链。
- `IndustryTabs` 用于切换产业概况、产业图谱、产业链强弱分析。
- `IndustryDetailContent` 根据当前产业链和当前页签渲染对应正文。
- `BackToIndustryList` 用于返回主页中的产业链列表或区域规划列表。
- `IndustryMap` 用于渲染产业链树状图谱、节点展开收起、节点 hover 公司浮层和右侧趋势图表。
- `CompanyProfileDrawer` 用于在点击相关公司后展示公司简介、基础信息、产业归属和企业评分。
- `IndustryStrengthAnalysis` 用于渲染产业链强补延图谱，包含节点图例、建议环节开关和强弱节点链路图。

公司详情页内部布局建议：

```text
CompanyDetailPage
├── CompanyHeader
│   ├── CompanyLogo
│   ├── CompanyTitle
│   ├── CompanyTags
│   ├── DownloadReport
│   └── FollowCompany
├── CompanyPrimaryTabs
├── CompanySecondaryTabs
└── CompanyDetailContent
    ├── CompanyBasicInfo
    ├── CompanyCapitalMarket
    ├── CompanyOperationInfo
    ├── CompanyRisk
    ├── CompanyJudicialRisk
    ├── CompanyIndustryInfo
    ├── CompanyIntellectualProperty
    ├── CompanyNews
    ├── CompanyGraphCenter
    ├── CompanyEvaluation
    ├── CompanyPeerAnalysis
    └── CompanyEcommerceDetail
```

内容渲染说明：

- `CompanyHeader` 用于展示企业名称、经营状态、股票信息、企业资质标签、报告下载和关注入口。
- `CompanyPrimaryTabs` 用于切换公司详情一级模块。
- `CompanySecondaryTabs` 用于展示当前一级模块下的二级分类，例如基本信息下的企业基本信息、股东信息、主要人员等。
- `CompanyDetailContent` 根据当前公司、一级页签和二级页签渲染对应正文。

## 交互流程

```text
用户打开系统
  ↓
进入登录页 /login
  ↓
登录成功
  ↓
进入主页 /home
  ↓
点击首页右侧产业链卡片或产业链标签
  ↓
进入产业详情页 /industries/:industryChainId
  ↓
在产业图谱中悬浮树子节点查看相关公司
  ↓
点击相关公司打开公司简介抽屉
  ↓
点击查看企业洞察详情进入公司详情页 /companies/:companyId
  ↓

  ↓

  ↓

  ↓

```

## 开发说明

- 优先保证登录、路由跳转、公共头部和三个业务页面骨架可用。
- 项目使用 TypeScript 完成，页面组件使用 `.tsx`，业务数据、接口类型和工具函数使用 `.ts`。
- 全局状态管理统一使用 Zustand，仅存放登录态、用户信息、跨页面筛选条件、当前产业链和当前公司等共享状态。
- 页面数据可先使用 Mock 数据，后续再对接真实接口。
- 公共头部应抽离为独立组件，避免在主页、投管池、投后页面重复实现。
- 业务页面共享同一个 `MainLayout`，登录页保持独立布局。
- 路由参数需要定义类型，例如 `industryChainId`、`companyId`。
- 首页产业链卡片、区域规划中的产业链标签和产业详情页左上方选择器应使用同一份产业链基础数据，避免名称和 ID 不一致。
- 产业图谱建议使用结构化树节点数据维护，节点至少包含名称、层级、父节点、子节点和相关公司列表。
- 相关公司数据至少包含公司名称、标签、基础工商信息、所属产业链、所属行业、风险提示和评分摘要，用于支撑公司简介抽屉展示。
- 公司简介抽屉和公司详情页应使用同一个 `companyId`，避免抽屉展示公司与详情页公司不一致。
- 产业链强弱分析建议使用结构化节点和连线数据维护，节点至少包含名称、层级、强弱状态、建议类型和关联说明。
- 路由命名、组件命名应保持语义清晰，方便后续扩展企业画像、合同审核等业务能力。
