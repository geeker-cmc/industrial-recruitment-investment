# AI产业招投数据平台 —— 互动Demo开发说明书
### （南通产控集团 · 15分钟演示脚本版）

> **本文档用途**：作为开发人员的需求/提示词（Prompt）依据，交给 **Claude Fable 5** 生成一版可交互的网页Demo（HTML/React单页应用即可，无需真实后端，全部数据Mock）。文档已结合客户提供的产品原型截图（首页 / 产业分析 / 企业画像 / 投管池 / 投后监测）整理，开发人员可直接摘取"六、给Fable 5的Prompt模板"章节，粘贴给模型执行。

---

## 一、项目背景与目标

- **客户**：南通产控集团
- **场景**：面向集团领导/招商团队的产品演示，展示"AI驱动的产业招商与投后管理"全流程能力
- **演示时长**：15分钟，共5个环节，环环相扣、层层递进（从"选产业"→"选企业"→"投中风控"→"投后监测"→"出报告"）
- **核心卖点**：AI在每一步都承担"专业分析师"的角色——自动生成产业诊断、企业画像综合评价、法律条款审查、估值建议、尽调报告，全程"输入即产出"，体现平台的智能化与专业深度

## 二、15分钟演示脚本总览

| 环节 | 时长 | 核心页面 | AI能力展示点 |
|---|---|---|---|
| 1. 产业分析 | 3分钟 | 首页 → 产业详情页 | 全国产业分析、南通"强补延"诊断、关键技术领域图谱 |
| 2. 企业画像 | 3分钟 | 企业画像页 | 工商/司法/舆情/财务/技术/人才/产业链多维数据整合 + 综合评价打分 + 同业对标 |
| 3. 投中风控 | 3分钟 | 备选池 → 合同审核页 | AI梳理投资协议法规与格式条款、基于历史协议生成专属模板 |

## 四、分环节详细设计

### 环节1：产业分析（3分钟）

**交互流程**
1. 用户在首页搜索框输入产业名称（演示建议用南通支柱产业，如"船舶海工装备"）
2. 点击搜索后，页面出现"AI正在分析产业数据…"的加载态（建议做成分步骤的进度提示，如"检索全国产业数据 → 比对南通产业链 → 生成关键技术图谱"，营造"AI真的在思考"的现场感）
3. 展示三个Tab结果：
   - **全国该产业分析**：产业规模、增长趋势、代表性企业、区域分布地图
   - **南通该产业"强补延"诊断**：强项环节（南通已有优势）、薄弱环节（需要补链）、可延伸环节（延链方向），建议用"红黄绿"三色标签直观呈现
   - **关键技术领域**：技术图谱树状图 + "卡脖子"技术清单 + 相关专利/企业数量

### 环节2：定位目标企业 · 企业画像（3分钟）

**交互流程**
1. 从产业详情页的"代表企业/延链企业"列表中点击某家目标企业（示例：**"通海重工科技有限公司"**，为演示虚构名称，需在文档/Demo中注明"数据为演示目的虚构"）
2. 进入企业画像页，分模块展示：
   - 工商信息（注册资本、股权结构、成立年限）
   - 司法信息（涉诉记录、失信记录）
   - 舆情信息（近期新闻情感分布）
   - 财务信息（营收、利润、资产负债率趋势图）
   - 技术信息（专利数量、核心技术领域）
   - 人才信息（核心团队背景、研发人员占比）
   - 产业链关系（上下游企业关联图谱）
   - **综合评价**：AI打分（如"投资价值评分：82/100"）+ 评分依据摘要
   - 同业公司对比（雷达图或对比表，展示该企业在同业中的相对位置）

### 环节3：加入备选池 / 投中风控合同审核（3分钟）

**交互流程**
1. 在企业画像页点击"加入备选池"按钮，跳转/弹窗展示备选池列表（可展示多家已入池企业作为背景）
2. 点击"进入投中风控"，触发合同审核流程：
   - 上传（或选择模拟已上传的）投资协议
   - AI展示"法律法规与格式条款梳理结果"：逐条列出协议条款类型、对应法规依据、是否存在风险点（如"违反《公司法》第XX条相关表述"，仅作演示提示，无需真实法条引用，可用"某法规条款"占位）
   - 展示"基于历史协议生成专属模板"入口：点击后生成一份带有南通产控集团历史投资协议风格的专属模板（可Mock为几个关键条款的对比：通用模板 vs 专属生成模板）

**Mock数据结构示例**
```json
{
  "contractReview": [
    {"clause": "对赌条款", "riskLevel": "中", "note": "业绩承诺触发条件与行业惯例存在差异，建议调整"},
    {"clause": "股权回购条款", "riskLevel": "低", "note": "符合常规格式条款要求"},
    {"clause": "信息披露条款", "riskLevel": "高", "note": "缺少定期披露频率约定，建议补充"}
  ],
  "customTemplate": {
    "basedOnHistoryCount": 23,
    "highlightClauses": ["投后管理权条款", "分期出资条款", "反稀释条款"]
  }
}
```

**话术要点**：强调"AI不是简单的合同比对工具，而是结合集团历史投资协议沉淀出的'专属知识'"。

### 环节4：投后监测（3分钟）

**交互流程**
1. 进入投后监测仪表盘，展示已投企业（沿用"通海重工科技"或另设一家已投企业）的指标变化趋势（财务、舆情、司法等指标随时间的曲线/柱状图）
2. 展示AI自动生成的"新一轮融资估值建议"卡片（如"建议估值区间：3.2亿-3.6亿元，较上轮增长18%，依据：营收增速、同业估值倍数"）
3. 展示部分指标的风控预警（如"舆情负面占比环比上升5%，建议关注"）与管理动作建议（如"建议约谈管理层"）

**Mock数据结构示例**
```json
{
  "monitoring": {
    "financialTrend": {"quarters": ["25Q2", "25Q3", "25Q4", "26Q1"], "revenue": [0.5, 0.55, 0.62, 0.7]},
    "riskAlerts": [
      {"indicator": "舆情负面占比", "change": "+5%", "level": "关注"},
      {"indicator": "核心专利到期", "change": "1项将于6个月内到期", "level": "提醒"}
    ]
  },
  "valuationSuggestion": {
    "range": "3.2亿-3.6亿元",
    "growthVsLastRound": "18%",
    "basis": ["营收增速25%", "同业平均PS倍数4.2倍", "技术壁垒溢价"]
  }
}
```

**话术要点**：突出"投后不是简单的报表汇总，而是AI主动预警+主动建议"，体现平台从"投前"延伸到"投后"的完整闭环能力。

---

### 环节5：报告生成（3分钟，可与其他环节穿插演示）

**交互流程**
1. 在企业画像页或投后页点击"生成投资能力画像报告"/"生成企业尽调报告"按钮
2. 展示"AI正在生成报告…"的加载态（模拟逐段生成效果）
3. 展示生成的报告大纲预览（可折叠展开每个章节）
4. 演示"补充数据后实时重新生成"能力：手动修改/补充一项数据（如财务营收数字），点击"重新生成"，展示报告中对应段落实时更新（哪怕只更新1-2句话，也要让评委看到"数据变了，结论也跟着变"）

**话术要点**：这是全场最重要的"收尾亮点"——证明系统不是"一次性生成的PPT"，而是"活的、可迭代的分析引擎"。

## 五、技术实现建议（给开发人员）

1. **技术栈**：建议使用单页 HTML + 少量JS（或React单文件组件），无需后端，所有数据以JSON Mock方式内置在前端代码中，便于现场演示时快速修改数据
2. **"AI生成感"的还原**：所有"AI分析/生成"环节都应做"渐进式呈现"效果——例如打字机效果逐字输出结论文字、进度条+文案切换的加载动画（"正在检索全国产业数据…" → "正在比对南通产业链…" → "生成完成"），这是本Demo能否让人"感觉AI真的在工作"的关键，务必做足
3. **导航方式**：建议做成"面包屑+步骤条"的形式（产业分析 → 企业画像 → 投中风控 → 投后监测 → 报告生成），方便演示者跳转、回退，不必强制线性
4. **图表建议**：产业规模/趋势用折线图，强补延诊断用三色标签或雷达图，同业对比用雷达图或横向条形图，投后指标趋势用折线图+预警标签
5. **示例产业/企业数据**：建议至少内置1-2个南通本地特色产业（船舶海工装备、高端纺织、新材料、电子信息均可）及对应虚构目标企业，所有企业名称、财务数据均为演示虚构，页面角落建议加一行小字"演示数据，非真实企业信息"
6. **响应式**：演示大概率在投影/大屏上进行，建议按 1440px 宽度以上桌面端优化，不必兼容移动端

## 七、演示注意事项

- 演示前务必确认现场网络/投影分辨率，Demo建议本地离线运行（纯前端HTML文件），避免依赖外部接口
- 各环节加载动画时长建议控制在2-4秒，太短显不出"AI在计算"，太长影响15分钟节奏
- 环节5的"补充数据实时重新生成"是全场记忆点，建议提前彩排，确保修改的数据字段与展示文案联动正确
- 所有企业名称、财务数据、合同条款均为演示虚构内容，正式对外材料中需明确标注，避免引发真实企业对号入座的误会
