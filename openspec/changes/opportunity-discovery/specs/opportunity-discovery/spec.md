## 新增需求

### 需求:工作台推荐商机入口

工作台必须提供“推荐商机”模块，展示基于当前用户投资风格生成的 3~5 家推荐企业，并提供“更多”入口进入发现企业列表。

#### 场景:工作台展示推荐企业
- **当** 用户登录后进入工作台，且存在阶段为“投”或“管”的项目企业样本
- **那么** 系统必须展示推荐商机模块，并显示 3~5 家候选企业的名称、所属产业链、推荐度、匹配标签和推荐理由
- **那么** 每家企业必须提供“查看画像”操作，且“更多”必须跳转到 `/opportunity-discovery`

#### 场景:没有投资样本时展示降级推荐
- **当** 用户进入工作台，且没有阶段为“投”或“管”的项目企业样本
- **那么** 系统必须基于南通重点产业、产业链匹配和企业综合评价生成推荐结果
- **那么** 推荐模块必须标明推荐来源为重点产业或其他降级规则，不得展示空白模块

### 需求:投中投后投资风格样本

系统必须将投资项目生命周期阶段为“投”和“管”的项目企业纳入当前用户投资风格样本，阶段为“募”的项目企业不得作为已投资风格样本。

#### 场景:提取投资风格
- **当** 系统生成用户投资风格
- **那么** 必须从阶段为“投”或“管”的项目企业中聚合产业链、企业标签、主营业务、区域、综合评价和科创评价特征
- **那么** 系统必须保留样本企业数量和主要偏好维度，用于在发现企业页面说明推荐依据

#### 场景:排除已纳入样本的企业
- **当** 系统生成候选企业列表
- **那么** 已作为“投”或“管”样本的企业必须默认从新的推荐候选中排除
- **那么** 已存在投资项目的企业必须显示或保留可识别的排除状态，禁止在默认推荐中重复出现

### 需求:多维筛选分组

发现企业页面必须提供分组折叠的多维筛选区。每个分组必须支持“全部”和多项复选；筛选分组至少包括政府奖励、认定认证、科技类认证、产业链与行业、区域、企业规模与发展阶段、投融资特征、经营与风险、技术与人才。

#### 场景:展开和收起筛选分组
- **当** 用户进入发现企业页面
- **那么** 系统必须展示筛选分组标题和当前已选数量
- **当** 用户点击某个分组的展开或收起操作
- **那么** 系统必须只切换该分组的选项可见状态，不得清除其他分组已选条件

#### 场景:选择认证类条件
- **当** 用户展开政府奖励、认定认证或科技类认证分组
- **那么** 系统必须提供分级或类型化选项，包括国家级、省级、市级、专项奖励、专精特新、专精特新“小巨人”、科技小巨人、瞪羚企业、独角兽企业和隐形冠军等选项
- **那么** 用户必须能够在同一分组内选择多个条件

### 需求:多维筛选组合语义

系统必须在用户确认查询时应用筛选条件。同一分组内多个选项按“或”匹配，不同分组之间按“且”组合；选择某个分组的“全部”必须表示该分组不限制候选企业。

#### 场景:组合多个分组查询
- **当** 用户选择“专精特新”和“高新技术企业”，并同时选择“电子信息产业链”和“南通市”后点击“确认查询”
- **那么** 结果必须匹配认证分组中的任一条件、产业链分组中的任一条件，并且同时满足区域条件
- **那么** 页面必须更新企业结果数量、已选条件摘要和列表内容

#### 场景:清空筛选条件
- **当** 用户点击“清空条件”
- **那么** 所有分组的选项必须恢复为未选择状态，已选条件摘要必须清空
- **那么** 系统必须恢复到当前推荐策略下的未筛选候选列表

### 需求:推荐策略与推荐解释

系统必须将推荐策略与筛选条件分开管理。推荐策略至少包括投资风格相似、产业链补链、南通重点产业、高成长优选和综合推荐；筛选完成后，系统必须按当前策略对候选企业排序，并展示可解释的推荐来源和匹配理由。

#### 场景:切换推荐策略
- **当** 用户在发现企业页面切换推荐策略
- **那么** 系统必须保留已确认的筛选条件，并按照新策略重新排序结果
- **那么** 每条企业记录必须更新推荐来源、匹配维度或推荐理由

#### 场景:查看推荐依据
- **当** 用户查看企业列表中的推荐结果
- **那么** 系统必须展示推荐度、推荐来源、匹配产业链、匹配标签和自然语言推荐理由
- **那么** 推荐度必须作为排序和参考信息，禁止被展示为已经完成投资决策

### 需求:发现企业列表与企业画像衔接

系统必须新增“发现企业”列表页面，并提供企业搜索、推荐策略、多维筛选和结果表格。企业名称和查看画像操作必须复用现有企业详情路由 `/companies/:companyId`。

#### 场景:进入发现企业列表
- **当** 用户点击工作台推荐商机模块的“更多”
- **那么** 系统必须跳转到 `/opportunity-discovery`
- **那么** 页面必须展示策略区、多维筛选区和企业结果表格

#### 场景:打开企业画像
- **当** 用户点击企业名称或“查看画像”
- **那么** 系统必须打开对应的企业详情页
- **那么** 企业详情页必须继续提供工商、司法、舆情、财务、技术、人才、产业链、综合评价和同业公司信息

### 需求:推荐企业进入项目流程

系统必须支持从发现企业列表或企业详情发起“加入备选池”和“新建项目”。新建项目必须带入企业 ID、推荐来源和推荐理由，但必须由用户确认提交后才创建项目。

#### 场景:从推荐企业新建项目
- **当** 用户在发现企业列表中点击某企业的“新建项目”
- **那么** 系统必须进入项目管理的新建项目表单，并预填企业、推荐来源和推荐理由
- **当** 用户补充项目名称、目标金额和负责人后提交
- **那么** 系统必须创建项目，并使项目能够继续进入尽调、估值、投决、合同、交割和投后管理流程

#### 场景:只查看不创建项目
- **当** 用户仅点击企业名称或查看画像
- **那么** 系统不得自动创建项目或改变项目状态
- **那么** 用户必须显式点击新建项目并提交表单后，项目才可以进入项目管理台账

### 需求:产业洞察流程保持不变

本变更不得修改产业洞察页面现有的产业选择、产业详情、产业链图谱、产业链强弱分析和企业分布行为，也不得新增顶部一级导航菜单。

#### 场景:访问产业洞察
- **当** 用户从顶部导航进入产业洞察或从现有入口访问产业详情
- **那么** 系统必须继续使用原有产业洞察路由和页面结构
- **那么** 推荐商机功能不得改变产业洞察的核心交互和展示内容
