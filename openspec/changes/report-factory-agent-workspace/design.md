## 上下文

当前 `/document-management` 路由复用了 `BusinessManagement` 的文档管理模块，页面主要提供文档列表、筛选、上传、预览和归档。现有 `/agents` 路由则是尽调和合同资料解析台账，负责上传文件后查看解析状态，两者都不是连续的对话式工作区。

本变更只替换报告工厂页面，不改变顶部导航名称、不改变 hash 路由、不修改项目、客户、风险和企业详情页面，也不改变现有智能体解析台账。应用是 React + TypeScript + Vite，使用 Ant Design、React Router 和 Zustand，当前没有真实 AI 服务，因此本期需要提供可交互的 mock 对话能力。

## 目标 / 非目标

**目标：**

- 将 `/document-management` 渲染为报告工厂智能体对话工作区。
- 左侧提供“新建任务”“技能”和历史对话列表，历史对话可恢复完整消息。
- 右侧提供对话正文、当前技能、上下文信息、文件输入和发送交互。
- 提供尽调、合同审核、投后服务、基金项目适配、资产配置、招商会谈、产业数据、企业落户、企业雷达等预置技能。
- 支持技能详情查看、启停和自定义技能配置。
- 在没有后端服务时，通过 mock 响应模拟技能对话和结构化结果。
- 对话和技能配置在浏览器刷新后仍可恢复，保持演示流程连续。

**非目标：**

- 不保留报告工厂旧的文档列表、高级筛选、上传文件、预览和归档页面。
- 不实现真实的大模型调用、流式输出、账号级云端同步或权限服务。
- 不把对话结果回写项目、企业、风险或文档模块。
- 不替换 `/agents` 中现有的尽调、合同资料解析任务台账。
- 不新增顶部一级路由，不修改既有业务模块的页面排版。

## 决策

### 1. 使用独立页面替换旧模块复用

新增 `src/pages/ReportFactory` 页面，并将 `/document-management` 的路由元素改为该页面。这样可以彻底移除报告工厂的旧文档管理界面，同时保留 `BusinessManagement` 供项目、客户和风险模块使用。旧的 `DocumentRecord` 数据结构继续保留，因为项目详情和工作台统计仍可能读取它。

备选方案是继续在 `BusinessManagement` 内增加 `moduleKey` 分支。该方案会让通用表格组件继续携带文档表格逻辑，页面边界不清晰，后续维护和样式隔离成本更高，因此不采用。

### 2. 使用单一 Zustand 工作区状态

新增报告工厂专用状态，至少包含：

- 历史对话列表和当前对话 ID；
- 消息列表、消息角色、时间、附件和结构化输出；
- 当前技能 ID；
- 技能目录、启用状态和自定义技能配置；
- 新建、切换、重命名、归档、删除对话的方法；
- 发送消息、追加 mock 回复和保存技能的方法。

使用 Zustand 的 `persist` 中间件保存演示数据到 `localStorage`。状态只服务于报告工厂页面，不复用项目状态，也不向其他业务模块写入数据。

### 3. 左侧采用“操作区 + 历史对话区”结构

左侧固定为窄栏：顶部放置“新建任务”和“技能”，下方按最近更新时间展示历史对话。点击“新建任务”时创建空白会话并清空右侧消息；点击历史对话时恢复该会话；对话项提供重命名、归档和删除操作。

任务管理中的“任务”仅代表一次智能体对话，不代表项目管理中的投资项目或后台待办任务。列表项展示对话标题、使用技能、最后更新时间和最近一条消息摘要。

### 4. 技能管理在右侧内容区切换展示

点击左侧“技能”后，右侧从对话正文切换为技能管理页面，左侧历史对话保持不变。技能管理分为系统技能和自定义技能两部分，以紧凑卡片展示名称、描述、启用状态和“使用技能”操作。

点击技能卡片可以查看技能详情，包括技能说明、输入方式、输出方式和可访问的上下文类型。系统技能只允许启用或停用；自定义技能额外支持编辑和删除。点击“使用技能”后创建一个新对话，并将该技能设置为当前技能。

### 5. 自定义技能使用表单配置

“添加技能”打开 Ant Design `Modal` 或 `Drawer`，配置项包括技能名称、描述、提示词、输入方式、输出格式和可用上下文。保存后加入自定义技能列表并默认启用。配置只影响本地 mock 对话行为，不执行外部系统操作。

### 6. 对话使用确定性的 mock AI 适配器

右侧对话区包含：当前对话标题、当前技能标签、消息列表、附件列表、文本输入框和发送按钮。发送后先追加用户消息，再根据当前技能生成一条结构化 mock 回复，例如摘要、关注点、分析结论和下一步问题。不同技能使用不同的响应模板，使演示中可以看出技能切换带来的差异。

文件上传只保存文件名、类型和大小等前端元数据，不向服务器上传，也不承诺真实解析。输入为空时禁止发送；生成回复期间显示处理中状态，完成后自动保存会话更新时间。

### 7. 保持现有页面风格和路由边界

复用现有 Ant Design 组件、颜色变量、按钮和表单风格，新增页面使用独立的 `report-factory-*` CSS 类，避免污染旧的 `management-*` 和 `agent-*` 样式。顶部导航仍显示“报告工厂”，hash 地址仍为 `#/document-management`。

## 风险 / 权衡

- [本地存储容量有限] → 限制历史对话数量和单条消息附件元数据大小，超限时提示用户清理历史对话。
- [mock 回复与真实 AI 能力有差距] → 在代码中隔离对话响应适配器，后续只替换适配器，不改变页面状态和交互协议。
- [用户可能混淆历史对话与项目任务] → 左侧明确使用“历史对话”标题，并在页面说明中标注任务代表一次对话。
- [删除旧页面后用户找不到文档归档入口] → 保留项目详情和现有业务数据结构；本变更不删除其他页面中的文档关联能力。
- [技能配置过于自由导致演示结果不稳定] → 系统技能使用固定配置，自定义技能只改变名称、说明和 mock 输出标签，不执行任意脚本。

## 迁移计划

1. 新增报告工厂页面、Zustand 状态和 mock 数据。
2. 将 `/document-management` 路由从 `BusinessManagementPage moduleKey="document"` 切换到新页面。
3. 保留 `BusinessManagement` 的文档模块代码和共享数据，确保其他页面不受影响。
4. 验证新建对话、切换历史、技能选择、技能配置、消息发送和刷新恢复流程。
5. 如需回滚，仅恢复路由元素即可，不需要迁移或删除本地业务数据。
