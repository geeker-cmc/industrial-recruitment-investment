## 新增需求

### 需求:报告工厂展示对话式工作区
`/document-management` 路由必须展示报告工厂智能体对话工作区，页面必须包含左侧操作与历史对话区域，以及右侧对话正文区域；页面禁止展示原报告工厂的文档列表、高级筛选、文档上传、文档预览和归档管理界面。

#### 场景:进入报告工厂
- **当** 用户访问报告工厂菜单对应的 `/document-management` 路由
- **那么** 系统显示“新建任务”“技能”“历史对话”和对话正文区域

#### 场景:旧文档管理功能不再显示
- **当** 用户进入报告工厂页面
- **那么** 页面不显示文档管理表格、高级筛选、旧的上传文件按钮、文档预览入口和文档归档入口

### 需求:用户可以创建新的对话任务
系统必须允许用户通过左侧“新建任务”开始一个新的空白对话，并将该对话作为一条历史对话保存；对话任务必须区别于项目管理中的投资项目和后台待办任务。

#### 场景:创建空白对话
- **当** 用户点击“新建任务”
- **那么** 系统创建新的对话并清空右侧当前消息，同时将新对话设为当前对话

#### 场景:首次发送后保存对话
- **当** 用户在新对话中发送第一条消息
- **那么** 系统将该对话保存到历史对话列表，并根据第一条消息生成可读的对话标题或摘要

### 需求:用户可以管理历史对话
系统必须在左侧“历史对话”区域按照最近更新时间展示对话；每条对话必须显示标题、当前技能、最近更新时间或消息摘要，并必须支持打开、重命名、归档和删除。

#### 场景:恢复历史对话
- **当** 用户点击历史对话列表中的一条记录
- **那么** 系统在右侧恢复该对话的完整消息、附件元数据和当前技能

#### 场景:重命名历史对话
- **当** 用户对历史对话执行重命名并提交非空名称
- **那么** 系统更新列表中的对话标题，并保留原有消息内容

#### 场景:归档历史对话
- **当** 用户对历史对话执行归档
- **那么** 系统将对话标记为已归档，并从默认的进行中列表中隐藏，但不得删除对话内容

#### 场景:删除历史对话
- **当** 用户确认删除历史对话
- **那么** 系统从历史列表和本地对话状态中移除该对话；如果删除的是当前对话，系统必须切换到空白对话

### 需求:当前对话可以使用选定技能
系统必须在当前对话顶部显示当前技能；当用户从技能管理中选择技能开始任务时，系统必须创建或切换到使用该技能的新对话。

#### 场景:显示当前技能
- **当** 当前对话已设置技能
- **那么** 对话头部显示技能名称和技能描述摘要

#### 场景:更换技能开始新对话
- **当** 用户在技能管理中点击某个启用技能的“使用技能”
- **那么** 系统创建新的空白对话并将该技能设置为当前技能，不得覆盖已有历史对话

### 需求:用户可以提交文本并获得智能体回复
系统必须提供文本输入框和发送按钮；用户发送非空文本后，系统必须先显示用户消息，再根据当前技能追加一条确定性的 mock 智能体回复；回复必须在当前对话中展示，不得修改其他业务模块数据。

#### 场景:发送有效问题
- **当** 用户输入非空问题并点击发送
- **那么** 系统追加用户消息、显示处理中状态，并最终追加包含摘要、关注点或建议的智能体回复

#### 场景:禁止发送空消息
- **当** 用户未输入文本且未添加附件时点击发送
- **那么** 系统禁止发送并提示用户输入问题或添加资料

#### 场景:对话结果不回写业务模块
- **当** 智能体生成回复或结构化结果
- **那么** 系统只更新当前对话内容，不创建或修改项目、企业、风险和文档记录

### 需求:用户可以向当前对话添加文件上下文
系统必须允许用户向当前对话添加文件，并至少保存文件名、文件类型和文件大小等前端元数据；本期禁止将文件上传到后端或声称已经完成真实文件解析。

#### 场景:添加文件后发送问题
- **当** 用户选择一个或多个文件并输入问题发送
- **那么** 系统在当前消息或附件区域展示文件元数据，并将文件作为 mock 回复的上下文提示

#### 场景:取消或移除文件
- **当** 用户在发送前移除附件
- **那么** 系统从当前对话待发送上下文中移除该文件，且不影响历史消息

### 需求:对话状态可以在刷新后恢复
系统必须使用现有 Zustand 方案保存当前对话、历史对话、消息、附件元数据和当前技能；浏览器刷新后必须恢复最近一次工作区状态。

#### 场景:刷新浏览器恢复工作区
- **当** 用户已经创建对话并发送消息后刷新浏览器
- **那么** 系统恢复历史对话列表、当前对话、消息内容和当前技能

## 移除需求

### 需求:报告工厂旧文档管理页面
**Reason**: 报告工厂改造成对话式智能体工作区，旧文档列表和归档页面与新的交互模型重复。
**Migration**: 项目详情和其他业务模块继续保留所需的文档数据结构；报告工厂路由改为使用对话工作区。
