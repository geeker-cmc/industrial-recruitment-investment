## 新增需求

### 需求:系统必须提供预置技能目录
技能管理页面必须提供以下预置技能：尽调 Skill、合同审核 Skill、投后服务 Skill、基金项目适配 Skill、资产配置 Skill、招商会谈 Skill、产业数据 Skill、企业落户 Skill和企业雷达 Skill；每个技能必须显示名称、用途描述和启用状态。

#### 场景:查看系统技能
- **当** 用户点击左侧“技能”
- **那么** 右侧显示技能管理页面，并列出全部预置技能及其描述和状态

#### 场景:搜索技能
- **当** 用户在技能管理页面输入技能名称或关键词
- **那么** 系统只显示名称或描述匹配的技能

### 需求:用户可以查看技能详情
系统必须允许用户点击技能卡片查看技能详情；详情必须包含技能说明、提示词摘要、支持的输入方式、输出格式和可用上下文类型。

#### 场景:打开技能详情
- **当** 用户点击“尽调 Skill”或其他技能卡片
- **那么** 系统在右侧展示该技能的详细配置和使用说明，并保留左侧历史对话列表

### 需求:用户可以启用或停用技能
系统必须允许用户启用或停用技能；停用技能禁止出现在可使用技能选择中，但历史对话中已经使用该技能的记录必须保持可查看。

#### 场景:停用技能
- **当** 用户将已启用技能切换为停用
- **那么** 系统更新技能状态，并禁止用户通过“使用技能”创建该技能的新对话

#### 场景:恢复启用技能
- **当** 用户将停用技能切换为启用
- **那么** 系统允许该技能重新被选择，并保留原有技能配置

### 需求:用户可以添加自定义技能
技能管理页面必须提供“添加技能”入口；添加技能表单必须支持填写技能名称、技能描述、提示词、输入方式、输出格式和可用上下文；技能名称和描述不能为空。

#### 场景:成功添加自定义技能
- **当** 用户填写必填配置并点击保存并启用
- **那么** 系统将自定义技能加入技能列表，标记为已启用，并允许用户立即使用该技能创建对话

#### 场景:缺少必填配置
- **当** 用户未填写技能名称或技能描述就提交表单
- **那么** 系统阻止提交并显示对应字段的校验提示

#### 场景:取消添加技能
- **当** 用户关闭添加技能表单或点击取消
- **那么** 系统不新增技能，已存在的技能配置保持不变

### 需求:用户可以维护自定义技能
系统必须允许用户编辑和删除自定义技能；系统预置技能禁止删除，只允许启用或停用。

#### 场景:编辑自定义技能
- **当** 用户修改自定义技能的描述、提示词或输出格式并保存
- **那么** 系统更新该技能配置，并在后续新对话中使用最新配置

#### 场景:删除自定义技能
- **当** 用户确认删除自定义技能
- **那么** 系统从技能目录中移除该技能，但不得删除历史对话内容

### 需求:用户可以从技能详情创建对话
每个已启用技能必须提供“使用技能”操作；该操作必须创建新的空白对话并返回对话工作区，且新对话顶部必须显示所选技能。

#### 场景:使用启用技能
- **当** 用户点击已启用技能的“使用技能”
- **那么** 系统创建新的空白对话，设置当前技能并切换到对话正文

#### 场景:使用停用技能
- **当** 用户尝试使用停用技能
- **那么** 系统禁止创建新对话，并提示先启用该技能

### 需求:技能配置在刷新后保留
系统必须使用现有 Zustand 持久化方案保存自定义技能及启停状态；刷新浏览器后必须恢复技能目录和配置。

#### 场景:刷新后恢复技能
- **当** 用户添加自定义技能或停用系统技能后刷新浏览器
- **那么** 系统恢复自定义技能、编辑内容和启停状态
