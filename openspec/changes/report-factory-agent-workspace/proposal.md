## 为什么

当前报告工厂仍然是文档列表管理页面，无法提供连续的资料分析和智能对话体验。将报告工厂改造成面向投资管理人员的智能体工作台，可以把历史对话、业务技能、资料输入和结构化输出集中到一个工作区，降低投研、风控和投后场景的使用成本。

## 变更内容

- **BREAKING**：删除报告工厂现有的文档列表、高级筛选、上传文件、文档预览和归档管理页面。
- 保留顶部“报告工厂”菜单及 `/document-management` 路由，不改变系统整体导航结构。
- 新增对话式智能体工作台：右侧展示当前对话、消息记录、文件输入、技能选择和 AI 输出。
- 新增历史对话任务管理：左侧展示用户创建过的对话，支持新建、恢复、重命名、归档和删除。
- 新增技能管理：提供尽调、合同审核、投后服务、基金项目适配、资产配置、招商会谈、产业数据、企业落户、企业雷达等技能。
- 新增技能详情和技能配置能力：支持配置技能说明、提示词、数据访问范围、输入方式、输出格式和可执行动作。
- 对话结果仅在当前智能体工作区内展示和保存，不回写项目、企业、风险或文档模块。
- 保留现有“智能体”路由中的尽调和合同资料解析台账，报告工厂作为其上层的通用对话入口。

## 功能 (Capabilities)

### 新增功能

- `agent-conversation-workspace`: 提供历史对话、新建对话、消息交互、资料上传、技能选择和结构化 AI 输出。
- `agent-skill-management`: 提供系统技能展示、技能详情、技能启停和自定义技能配置。

### 修改功能

无。

## 影响

- 路由：`/document-management` 将从文档管理页面切换为独立的报告工厂智能体工作台，顶部菜单名称保持不变。
- 页面：新增报告工厂工作区页面，移除该路由对 `BusinessManagement` 文档模块的复用。
- 状态：需要新增对话、历史任务、当前技能和技能配置的前端状态管理，沿用现有 Zustand 方案。
- 数据：对话可关联用户输入的企业、项目或资料名称作为上下文，但不修改其他业务模块的数据。
- 依赖：优先使用现有 React、Ant Design、React Router 和 Zustand，不引入新的 UI 框架。
